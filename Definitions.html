• Branch: Allows you to work on different features
  • Clone: Creates a copy of the repository
  • Commit: Records the changes and sends a descriptive text
  • Fetch: Download changes to your local machine from your repository
  • GIT: A control system used to track any sorts of changes from the source
  • GitHub: A platform being able to share repository
  • Merge: Changing one branch to another
  • Merge Conflict: Git cannot resolve any difference between the two commits
  • Push: Uploading the repository changes to the remote repository
  • Pull: Download and make changes from remote repository to the local one
  • Remote: A repository that is a host of the server
  • Repository: A place where your project files and also the history are stored
